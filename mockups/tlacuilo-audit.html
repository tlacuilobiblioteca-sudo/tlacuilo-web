uso correcto vs uso roto

  • en /biblioteca, /videoteca, /artoteca, /calendario, /mi-tlacuilo, /admin/* — bien usados via tokens (bg-bg, text-text, border-rule, etc.)
  • dirty yellow solo aparece en eyebrows y chips activos · respeta "nunca dominante"
  • /login y /registro hardcodean #15151d (tinta ok) pero usan #9091c4 que NO está en paleta v2 · es periwinkle v1 deprecated
  • cuando hagas refactor de auth, reemplazar #9091c4 con var(--color-periwinkle) (#9D9BC8) o tokenizar todo

qué tocar antes del launch de junio

recomendaciones priorizadas

de más urgente a más nice-to-have. P0 = bloqueador de coherencia, P1 = mejora notable, P2 = polish futuro.

P0
auth · login/registro

migrar /login y /registro a paleta v2 + tokens

son las páginas más rotas del sistema. usan #9091c4 (color v1 deprecated) y font-futura (no cargada). un usuario nuevo entra por aquí y ve OTRA app. refactor a un AuthShell component que use bg-bg, text-text, font-mono. mantén el estilo terminal del prompt si te gusta, pero con los colores correctos.

P1
consistencia · headers

unificar el patrón de header de página

mi-tlacuilo y calendario usan eyebrow (DM Mono dirty) + h1 grande (Jost Light) y se ven 10/10. biblioteca solo dice "Página 1 de 47", videoteca/artoteca usan "videoteca · página X". aplicar mi-tlacuilo pattern a las 3 tecas: eyebrow "biblioteca/videoteca/artoteca" + h1 "Catálogo" / "Películas" / "Piezas".

P1
artoteca · placeholders

fallback de pieza sin foto: ficha técnica grande tipo wall label

35 de 113 piezas no tienen foto. el placeholder actual (cuadrado con título serif italic) es ok pero impersonal. para piezas de arte funciona mejor mostrar grande la metadata (título · artista · año · técnica · medidas) tipo wall label de museo. comunica que la pieza existe aunque no haya imagen.

P1
artoteca · sidebar

colapsar lista de 81 artistas o agregar búsqueda

la sidebar de artoteca tiene 81 chips. visualmente correcto pero abrumador. opciones: collapse/expand con "ver más", agrupar artistas con 1 sola obra en chip "otros", o agregar input de búsqueda arriba de los chips.

P1
tono · voz

decidir línea editorial: cuándo "terminal `>_`" y cuándo "clean Jost"

la app tiene dos personalidades visuales que coexisten: terminal hacker (login, registro, checkout, buscar, manifesto) y clean editorial (landing, biblioteca, artoteca, videoteca, mi-tlacuilo). ambas funcionan pero el quiebre se nota al navegar. propuesta: clean editorial en TODO lo público (catálogos, perfil, calendario), terminal solo en momentos íntimos (login, checkout post-confirmación, "tu identidad solo tú la ves").

P2
admin · consolidación

fusionar /admin/portadas dentro de /admin/libros

desde el refactor del editor rico, /admin/portadas hace casi lo mismo. en /admin/libros agregar tabs/filtros: todos · sin portada · pendientes · joyas. menos páginas, menos confusión sobre cuál usar.

P2
item detail · especialización

layout diferenciado por teca para detalle de item

actualmente /biblioteca/[id] sirve para libros, vhs y piezas de arte usando el mismo template. para artoteca el cover debería ser grande (object-contain, aspect natural). para vhs podría tener trailer/sinopsis TMDB. layout adaptativo por teca.

P2
buscar · filtros

agregar filtros a /buscar: teca, disponibilidad, categoría

el RPC search_libros devuelve todo. cuando el catálogo crezca, filtros chips arriba del input ayudan: [todo] [biblioteca] [artoteca] [videoteca] · [disponible] · categorías.

P2
calendario · interacción

click en día con evento abre popover/drawer en el grid

actualmente los eventos se listan abajo del grid. duplicación visual. mejor: hover en día muestra tooltip (ya está parcialmente), click abre drawer/popover con detalles. la lista debajo puede mantenerse como vista alternativa.

P2
data · sin fecha

chip "sin fecha" en videoteca y artoteca

las 12 Civilizaciones Perdidas (videoteca) y muchas piezas (artoteca) no tienen año. al filtrar por década quedan invisibles. agregar chip "sin fecha" como filtro explícito captura ese subconjunto.
